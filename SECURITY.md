# Security Policy

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, report them privately via
[GitHub Security Advisories](https://github.com/mteimoori/axe-hud/security/advisories/new).

You can expect an initial response within a reasonable timeframe. Once a fix is available, we
will coordinate disclosure.

## Scope

axe-hud is a development/QA tool intended to run in non-production environments. It must not be
enabled in production. Reports that depend on intentionally bypassing the production gate will be
assessed on a case-by-case basis.
